Getting started

While creating my first watch face using the Jetpack Watch Face API I discovered that there is no standard editor in the smartwatches, and that for each new watch face it is necessary to create a new editor.

As I intend to develop more watch faces using the Jetpack API, and I like the design and interaction mode of the WFS editor, I decided to create a module, with a similar interface, to standardize and simplify the creation of editors. I also decided to share this module, as a library, to help others.

This documentation guides on how to implement the library and how to use/customize it.

Add the dependecies

Add the jitpack repository to your settings.gradle file.


dependencyResolutionManagement {
    repositoriesMode.set(RepositoriesMode.FAIL_ON_PROJECT_REPOS)
    repositories {
        //...

        maven { url 'https://jitpack.io' } // <== ADD THIS LINE
    }
}

Add to your build.gradle app module file the following dependency:


implementation 'com.github.hms-douglas:wfe:1.0.1' //Always check for the lastest
Create the activity

Declare an editor activity on you AndroidManifest.xml file. The activity theme must be Theme.AppCompat.NoActionBar.


<activity
    android:name=".YouActivityClassNameHere"
    android:exported="true"
    android:theme="@style/Theme.AppCompat.NoActionBar">
    
    <intent-filter>
        <action android:name="androidx.wear.watchface.editor.action.WATCH_FACE_EDITOR"/>

        <category android:name="com.google.android.wearable.watchface.category.WEARABLE_CONFIGURATION"/>
        <category android:name="android.intent.category.DEFAULT"/>
    </intent-filter>
</activity>

Add to you watch face service tag the following meda data tag:


<meta-data
    android:name="com.google.android.wearable.watchface.wearableConfigurationAction"
    android:value="androidx.wear.watchface.editor.action.WATCH_FACE_EDITOR"/>

Create your YouActivityClassNameHere file. This activity/class must extend the WFEActivity class. Example:


public class YouActivityClassNameHere extends WFEActivity {

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
    }
}
Init

To create the editor you must call the initEditor() function, passing the parameters to define the design of the editor and the options screens you want it to render.

The initEditor() function must be called before the super.onCreate() of the activity you have created. Example:


public class YouActivityClassNameHere extends WFEActivity {

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        //...

        initEditor(listenableEditorSession, wfes, wfeol);

        super.onCreate(savedInstanceState);
    }
}

WFEColorListOption

Class responsible for rendering the basic color list option screen.

The user can swipe up and down to switch between each color item.

The list "scrollbar" will render up to 9 items, but more items will be displayed, if avaiable, while scrolling.

This option uses the ListUserStyleSetting to render the colors.

Each ListUserStyleSetting.ListOption will represent one item.

Each color MUST be set, as hex string, in the displayName for each ListUserStyleSetting.ListOption. To display more than one color for item, separate each hex string using commans.


// Example of ListUserStyleSetting.ListOption (not part of the editor/constructor).

new UserStyleSetting.ListUserStyleSetting.ListOption(
    new UserStyleSetting.Option.Id("optionIdHere"),
    getResources(),
    R.string.option_string_here, // <<<< FOR ONE COLOR "#FF0000" 
    null);

// OR

new UserStyleSetting.ListUserStyleSetting.ListOption(
    new UserStyleSetting.Option.Id("optionIdHere"),
    getResources(),
    R.string.option_string_here, // <<<< FOR N COLORS "#FF0000, #00FF00, ..." 
    null);

WFEDynamicColorPickerOption

Class responsible for rendering a color picker.

The user can swipe up and down to scroll through the picker. If the user swipe on the left half of the screen, the left scroller will move. If the user swipe on the right half of the screen, the right scroller will move.

This option uses the CustomValueUserStyleSetting to return the color picked. The color picked will be in string hex format (Example: "#FF0000").

The default UserStyleSetting.Id string for the CustomValueUserStyleSetting.CustomValueOption is "CustomValue".

*Only one CustomValueUserStyleSetting is allowed. This is a limitation of Wear OS. Therefore this option will get in conflict with WFEStaticColorPickerOption if both options are added to the editor. This option will also get in conflict with itself if more than one is added to the editor.

Annotation Type Name Description
@NonNull String label Label to be displayed at the top of the screen.

Extra
Rotary input

After Wear OS 4, Google is being obsessed with their guidelines for Wear OS. One of the guidelines is the rotary input.

According to their documentation, every app must allow the user to scroll through screens and/or pickers using the rotary input. This guideline applies to watch faces editor/customizers as well. Therefore I implemented the rotary input feature in the WFE.

Using the rotary input the user can:

WFEButtonOption: toggle on/off;

WFEListOption: navigate between the items;

WFEColorListOption: navigate between the colors;

WFEDynamicColorPickerOption: move the picker up and down. To move the left picker, the user must tap on the left half of the screen. To move the right picker, the user must tap on the right half of the screen. By default, the right picker is focused;

WFEStaticColorPickerOption: navigate between the colors;

WFEComplicationOption: no actions available;

WFEDoubleRangeOption: increment/decrement the range value;

WFELongRangeOption: increment/decrement the range value;

WFELabelOption: scroll the text (if overflowed);

WFEButtonOption: no actions available;

WFEWatchfaceOption: no actions available.

*To scroll through the options screen the user must swipe left and right.

YouActivityClassNameHereExample 

//Example on how to write your "YouActivityClassNameHere" activity class
public class YouActivityClassNameHere extends WFEActivity {

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        final WFESettings settings = new WFESettings();

        // Setting some design/style and behavior of the editor;
        settings.setMainColor("#FF0000");
        settings.setRenderPreviewDelay(1000);
        settings.setVibrationFeedback(false);

        // Option list / Each item will render a screen
        final List<WFEOption> options = new ArrayList<>();

        // Adding some boolean screens
        options.add(new WFEBooleanOption(new UserStyleSetting.Id("show_secs"), "Show/hide seconds"));
        options.add(new WFEBooleanOption(new UserStyleSetting.Id("dig_style")));

        // Creating a basic list screen with some settings
        final WFEListOption fonts = new WFEListOption(new UserStyleSetting.Id("font"), "Font style");

        fonts.setDisplayOptionNameWhenSelected(true);
        fonts.setFastScroll(false); 

        // Adding the basic list screen
        options.add(fonts);

        // Adding a complication screen that allows the user to pick a complication for the ComplicationSlot.IDs 0 and 2
        options.add(new WFEComplicationOption(0, 2));

        // Adding another complication screen that allows the user to pick a complication for the ComplicationSlot.ID 1
        options.add(new WFEComplicationOption(1));

        // Adding a button screen
        options.add(new WFEButtonOption("Hey, I'm a button!", () -> {
            // Action to be executed when the user clicks the button
            Toast.makeText(this, "Clicked", Toast.LENGTH_SHORT).show();
        }));

        /*
        * Init the editor UI
        *
        * Must be called before super.onCreate()
        * It is NOT recommended to set the ListenableEditorSession to null like I did here!!!
        * If null, the application will try to get it automatically...
        */
        initEditor(null, settings, options);

        super.onCreate(savedInstanceState);
    }
}
